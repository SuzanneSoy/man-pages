NOME

debconf-set-selections - insere novos valores na base de dados do debconf  

SINOPSE

 debconf-set-selections ficheiro
 debconf-get-selections | ssh newhost debconf-set-selections
 

DESCRIÇÃO

debconf-set-selections pode ser usado para pré-semear a base de dados debconf com respostas, ou para modificar as respostas na base de dados. Cada questão será marcada como 'vista' para prevenir o debconf de perguntar a questão interactivamente.

Lê a partir dum ficheiro de um nome de ficheiro é fornecido, caso contrário a partir do stdin.  

AVISO

Use apenas este comando para semear valores de debconf para pacotes que irão ser ou estão instalados. Caso contrário você pode acabar com valores na base de dados para pacotes desinstalados que não desaparecem, ou com problemas piores que envolvem valores partilhados. É recomendado que isto apenas seja usado para semear a base de dados se a máquina original tiver uma instalação idêntica.  

FORMATO DE DADOS

Os dados são uma série de linhas. As linhas começadas com o caractere # são comentários. As linhas vazias são ignoradas. Todas as outras linhas definem o valor de uma questão, e deve conter quatro valores, cada um separado por um caractere de espaço em branco. O primeiro valor é o nome do pacote que possui a questão. O segundo é o nome da questão, o terceiro valor é o tipo de questão, e o quarto valor (até ao fim da linha) é o valor a usar para a resposta à questão.

Alternativamente, o terceiro valor pode ser ``visto''; então a linha precedente apenas controla se a questão está marcada como vista na base de dados do debconf. Note que para preceder os valores predefinidos de uma questão para marcar essa questão como vista, portanto para sobrepor o valor predefinido sem marcar a questão como vista, você precisa de duas linhas.

As linhas podem continuar para a próxima linha ao acabarem com o caractere ``\''.  

EXEMPLOS

 # Força a prioridade do debconf para crítica.
 debconf debconf/priority select critical

 # Sobrepõe o frontend predefinido para readline, mas permite ai utilizador seleccionar.
 debconf debconf/frontend select readline
 debconf debconf/frontend seen false
 

OPÇÕES

--verbose, -v
saída detalhada
--checkonly, -c
apenas verifica o formato do ficheiro de entrada, não guarda alterações na base de dados
