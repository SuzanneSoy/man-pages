NOME

dh_installmanpages - instalador de manuais ao estilo antigo (descontinuado)  

RESUMO

dh_installmanpages [debhelper opções] [ficheiro ...]  

DESCRIÇÃO

dh_installmanpages é um programa debhelper que é responsável por instalar automaticamente manuais (man pages) em usr/share/man/ nos directórios de compilação de pacotes.

Este é um programa estilo DWIM, com uma interface diferente do resto do debhelper. Está descontinuado, em vez deste, você é encorajado a usar o dh_installman(1).

dh_installmanpages examina o directório actual e todos os sub-directórios por nomes de ficheiros que se parecem com manuais. (Note que apenas são procurados ficheiros verdadeiros, os links simbólicos são ignorados). Usa o file(1) para verificar se os ficheiros estão no formato correcto. Depois, baseando-se nas extensões dos ficheiros, instala-os no directório de manuais correcto.

Todos os nomes de ficheiros especificados como parâmetros serão evitados pelo dh_installmanpages. Isto é útil se por predefinição instalar alguns manuais que você não deseja serem instalados.

Após o passe da instalação do manual, o dh_installmanpages irá ver se alguns dos manuais são links .so. Se forem, altera-os para links simbólicos.  

OPÇÕES

ficheiro ...
Não instale estes ficheiros como manuais, mesmo que eles se pareçam com ``man pages'' válidas.
 

BUGS

dh_installmanpages irá instalar os manuais que encontra em todos os pacotes que lhe diga para actuar, pois ele não sabe a qual pacote os manuais pertencem. Quase sempre isto não é o que realmente você quer (use -p para contornar isto, ou use o outro muito melhor programa dh_installman(1) em vez deste).

Os ficheiros que terminem em .man serão ignorados.

Ficheiros especificados como parâmetros que contêm espaços nos seus nomes de ficheiros não serão processados de modo apropriado.  

VEJA TAMBÉM

debhelper(7)

Este programa é parte do debhelper.
