NOMBRE

deb-old - Antiguo formato de paquete binario de Debian  

SINOPSIS

nombre-fichero.deb  

DESCRIPCIÓN

El formato .deb es el formato de fichero de paquete binario de Debian. Esta página de manual describe el formato antiguo, usado previamente a la versión 0.93 de Debian. Consulte el manual deb(5) para más detalles sobre el nuevo formato.  

FORMATO

El fichero está formado por dos lineas con información del formato en texto ASCII, seguido por dos ficheros ustar concatenados y comprimidos vía gzip.

La primera línea indica la versión del formato completada con 0 hasta cubrir 8 dígitos, y es 0.939000 para todos los archivos con formato antiguo.

La segunda línea es un número decimal (sin ceros precedentes) que indica el tamaño del primer fichero tar comprimido con gzip.

Cada línea finaliza con un único carácter de nueva línea.

El primer fichero tar contiene la información de control como un conjunto de ficheros comunes. El fichero control debe estar presente, ya que contiene la información de control básica.

In some very old archives, the files in the control tarfile may optionally be in a DEBIAN subdirectory. In that case, the DEBIAN subdirectory will be in the control tarfile too, and the control tarfile will have only files in that directory. Optionally the control tarfile may contain an entry for '.', that is, the current directory.

El segundo fichero tar comprimido con gzip es el archivo del sistema de ficheros, que contiene las rutas relativas al directorio raíz del sistema donde se instalará. Los nombres de ruta no tienen barras precedentes.
