deb-prerm - Präentfernungsbetreuerskripte eines Pakets  

ÜBERSICHT

prerm  

BESCHREIBUNG

Ein Paket kann eine Reihe von Präentfernungsaktionen mittels seiner Betreuerskripte durchführen, indem es eine ausführbare Datei prerm in seinem Steuerarchiv (d.h. DEBIAN/prerm während der Paketerstellung) hinzunimmt.

Das Skript kann in den folgenden Weisen aufgerufen werden:

prerm remove
Bevor das Paket entfernt wird.
old-prerm upgrade neue_Version
Vor einem Upgrade.
new-prerm failed-upgrade alte_Version neue_Version
Falls das obige upgrade fehlschlägt.
prerm deconfigure in-favour neues_Paket neue_Version

    [ removing altes_Paket alte_Version ] Bevor ein Paket dekonfiguriert wurde während eine Abhängigkeit aufgrund eines Konflikts ersetzt wird.
prerm remove in-favour neues_Paket neue_Version
Das Paket wird aufgrund eines Konflikts ersetzt.
